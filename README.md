# Attendance CLI Tool

Консольное приложение на C++ (C++20) для анализа посещаемости студентов. Проект реализует собственный JSON-парсер (DOM-модель), валидацию данных, генерацию отчетов и бенчмаркинг производительности.

## Особенности
- **Собственный парсер JSON:** Рекурсивный спуск, поддержка стандарта RFC 8259 (частично).
- **Надежность:** Обработка ошибок ввода, валидация типов данных, защита от переполнений.
- **Unicode:** Полная поддержка UTF-8 в консоли Windows и файлах (русские имена, пути).
- **Модульность:** Разделение на проекты (Main, Tests, Benchmark, Generator).

## Структура репозитория
- `src/` — Исходный код основного приложения.
- `include/` — Заголовочные файлы (`simple_json`, `attendance`, `utils`).
- `tests/` — Модульные тесты и генераторы данных.
- `data/` — Примеры входных файлов (генерируются автоматически или вручную).
- `docs/` — Документация, отчеты, план реализации.

## Сборка и запуск (Visual Studio 2022)

### Предварительные требования
- Visual Studio 2022 (MSVC).
- Стандарт C++20 (`/std:c++20`).
- Флаг компиляции `/utf-8` (для корректной работы с кириллицей).

### Инструкция
1. Откройте файл решения `.sln`.
2. В обозревателе решений доступно 4 проекта:
   - **Main:** Основное приложение (CLI).
   - **Tests:** Unit-тесты парсера.
   - **Generator:** Утилита для создания тестовых данных.
   - **Benchmark:** Специальная сборка для замеров скорости.
3. Установите **Main** как запускаемый проект (ПКМ -> Назначить запускаемым проектом).
4. Нажмите `F5` или `Ctrl+F5`.

> **Важно:** Рабочий каталог настроен на `$(SolutionDir)Lab_Final_09`, поэтому программа автоматически видит папку `data/`.

## Использование

### Интерактивный режим
Запустите приложение без аргументов.
```bash
./Main.exe
```
Вас встретит меню с возможностью выбора отчетов, фильтрации и сохранения.

### Аргументы командной строки
- `--help`: Справка.
- `--input <file>`: Указать путь к JSON файлу.
- `--student <name>`: Вывести отчет по студенту и выйти.
- `--bench`: Запустить тест производительности.

### Пример запуска
```powershell
# Генерация большого файла (если собран Generator)
./Generator.exe

# Запуск анализа
./Main.exe --input data/example_huge.json --bench
```# Laba
